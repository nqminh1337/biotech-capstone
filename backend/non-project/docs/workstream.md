# Workstream 2 - Back-end core services & APIs

## Purpose
To develop the core backend infrastructure that supports the mentoring platform’s essential services. This includes secure authentication, user and group data management, chat messaging, resource storage, and external system integrations, all built to scale and ensure data security.

## Detailed Design & Functions
**Authentication Service**:
Implement a passwordless login mechanism that issues short-expiry, single-use email codes for user sign-in. Includes token management and session security.

**User and Role Management**:
Build a comprehensive database schema that models users, their roles (student, mentor, supervisor, administrator), and relationships with groups and resources. Role-based permissions strictly govern access.

**Tracks and Groups Structure**:
Model mentoring tracks based on geographic regions and other criteria. Support dynamic creation and management of mentoring groups tied to these tracks.

**Chat Service**:
Develop a scalable chat backend capable of handling real-time messaging and file transfers within groups. The service may be custom-built or integrate with a provider, depending on evaluation.

**Resource Library Storage**:
Handle secure upload, storage, and retrieval of documents and resources with role-based visibility controls.

**API Endpoints**:
- Provide a robust RESTful API layer exposing endpoints for:
- User profile import/export (including syncing with Airtable profiles)
- Retrieving and managing group memberships and chat messages
- Account activation/deactivation and status management

**External Integrations**:
Enable synchronisation with Airtable for profile management and Humanitix for event data, ensuring data consistency across systems.

## Security & Scalability
All services enforce HTTPS, encrypted data storage, role-based access, secure file uploads, and audit logging. The backend architecture supports modular expansion and cloud deployment on AWS or Azure.

## Integration Points
This front-end consumes APIs provided by Workstream 2 for authentication, user data, chat messages, and resource management. It also displays matching and group data generated by Workstream 3.
